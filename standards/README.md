# Standards for Human-Centered MongoDB Education

This directory contains the pedagogical principles and design patterns for MongoDB labs targeting human learners.

## Files

- **[instructional-design-rulebook.md](instructional-design-rulebook.md)** — Core principles for effective MongoDB education. Read this first.
- **[lab-design-template.md](lab-design-template.md)** — Step-by-step template for creating a new lab (coming soon)
- **[rubric-for-labs.md](rubric-for-labs.md)** — How we evaluate lab quality before deploying to learners (coming soon)

## Quick Start

1. Read [instructional-design-rulebook.md](instructional-design-rulebook.md) — 10 min
2. Use [lab-design-template.md](lab-design-template.md) to outline your lab — 1–2 hours
3. Get peer feedback
4. Build the lab environment
5. Run a learner through it (feedback loop)

## Key Difference from `agent-training`

The `agent-training` repository's rulebook is optimized for agents: *explicit scaffolding, no ambiguity, mechanical grading*. This repository's rulebook will optimize for humans: *strategic struggle, social learning, formative feedback, real-world motivation*.

We'll adapt principles as needed, but many will be identical (e.g., backward design, three KLI types). The differences will be in *how we scaffold* and *when we give feedback*.
